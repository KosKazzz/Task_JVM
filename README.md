<h3>Загрузка и связывание</h3>
При первом запуске(в IDE) JvmComprehension.java c помощью javac(Java Compiler)
преобразуется в JvmComprehension.class. После этого IDE вызывает JvmComprehension.class,
ClassLoader загружает этот класс в основную память.
Учитывая , что в данном случае у нас JvmComprehension.class единственный класс в classpath,
его загружает Application Class Loader. Классы Maven загружает Platform Class Loader.
Cтандартные библиотеки Java (java.lang) загружает Bootstrap Class Loader.
После загрузки происходит связывание(Linking). На этом этапе происходит проверка корректности байт-кода
(Verification), выделение оперативной памяти под статические поля и инициализация их значениями
по умолчанию(Preparation).
<h3>Инициализация и выполнение</h3>
Происходит выхов метода main, для его выполнения в стеке выделяется фрейм.
1. Для переменной i ранее было выделено место в стеке и на этом этапе присвоено значение 1, находится в стеке.
2. Ранее было выдело место под переменную "о" ссылочного типа Object, теперь в куче(heap-eden) был создан объект типа Object
и ассоциированс с переменной "о".
3. Тут ситуация похожа на предыдущий пункт, объявляется ссылочная переменная ii типа Integer
в стеке. Integer  является классом с единственным полем типа int, этому полю и присваивается 
занчение "2". Хранится в куче(heap-eden).
4. Вызывается метод printALL(...), для него в стеке выделяется фрейм. Этому методу, в качестве 
параметров, передаются ранее инициализированные переменные.
5. В методе printALL(...) , аналогично п.3, объявляется и инициализируется ссылочна переменная uselessVar типа Integer
со значением "700"(хранится в куче(heap-eden)).
6. Создаётся новый фрейм в стеке и вызвается метод println() у переменной out типа PrintStream из класса System.
Которому передаются ссылки на объекты из кучи(heap) "ii" и "о" и значение i. 
7. На этом этапе GC удалят ссылку на uselessVar т.к. эта переменная не используется и вызывает,аналогично предыдущему 
пункту System.out.println() с параметром типа String, расположенным в куче(heap).

Прогамма завершается  и GC удаляет все ссылки и очищяется стек.
